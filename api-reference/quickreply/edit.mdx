---
title: "Editar Resposta Rápida"
api: "POST /quickreply/edit"
description: "Atualizar template de resposta rápida existente"
---

Atualiza um template de resposta rápida existente. Permite modificar atalho, tipo de mensagem e conteúdo do template.

## Sobre Edição de Respostas Rápidas

<AccordionGroup>
  <Accordion title="Atualizando Templates" icon="pen-to-square">
    Como modificar templates existentes.

    **Possibilidades:**
    - Alterar atalho (shortcut) do template
    - Mudar tipo de mensagem (texto para imagem, etc)
    - Atualizar conteúdo e mídia
    - Modificar legendas e descrições
    - Renomear arquivos de documentos

    **Limitações:**
    - Não é possível editar templates do WhatsApp (onWhatsApp=true)
    - ID do template não pode ser alterado
    - Atalho deve ser único entre seus templates
    - Validações de tipo continuam aplicadas

  </Accordion>

  <Accordion title="Identificação de Templates" icon="fingerprint">
    Como identificar o template a ser editado.

    **Identificador:**
    - Use o campo `id` retornado na criação
    - ID é único e imutável
    - Formato: `rb9da9c03637452`
    - Obtenha o ID listando templates existentes

    **Dica:**
    - Armazene IDs ao criar templates
    - Use endpoint de listagem para recuperar IDs
    - IDs não mudam durante a vida do template

  </Accordion>

  <Accordion title="Mudança de Tipo" icon="arrow-right-arrow-left">
    Convertendo entre tipos de mensagem.

    **Conversões permitidas:**
    - Texto → Mídia: adicione campo `file`
    - Mídia → Texto: remova campo `file`, mantenha `text`
    - Entre mídias: altere `type` e atualize `file`
    - Adicionar/remover legendas a qualquer momento

    **Atenção:**
    - Ao mudar para texto, campo `text` é obrigatório
    - Ao mudar para mídia, campo `file` é obrigatório
    - Tipo `document` aceita campo `docName` opcional
    - Valide URL ou base64 ao atualizar arquivo

  </Accordion>

  <Accordion title="Boas Práticas" icon="star">
    Recomendações para edição eficiente.

    **Organização:**
    - Mantenha histórico de versões dos templates
    - Documente mudanças significativas
    - Teste templates após edição
    - Atualize documentação interna

    **Performance:**
    - Edite em lote quando possível
    - Valide dados antes de enviar
    - Implemente retry para falhas temporárias
    - Monitore tempo de resposta

  </Accordion>
</AccordionGroup>

### Authorization

<ParamField path="api_key" type="string" required>
  Chave de autenticação da sua conexão, utilizada no header Authorization como
  Bearer Token. <br />
  Você encontra sua API Key no botão **API** da sua conexão no painel ZapClass. <br />
  A chave sempre começa com o prefixo `zc_`. <br />
  Exemplo: `zc_6FrX2qjM4bHT9pR6cC0sD8wLkG3nE2tZfV5uN1aP`
</ParamField>

### Body Parameters

<ParamField path="id" type="string" required>
  ID do template a ser editado. <br />
  Obtido na criação ou listagem. <br />
  Exemplo: `"rb9da9c03637452"`
</ParamField>

<ParamField path="shortCut" type="string">
  Novo atalho para o template. <br />
  Apenas letras, números e underscore. <br />
  Exemplo: `"saudacao_v2"`
</ParamField>

<ParamField path="type" type="string">
  Novo tipo da mensagem. <br />
  **Valores:** `text`, `image`, `video`, `audio`, `document` <br />
  Exemplo: `"image"`
</ParamField>

<ParamField path="text" type="string">
  Novo conteúdo de texto ou legenda. <br />
  Obrigatório se `type` for `text`. <br />
  Exemplo: `"Olá! Nosso horário foi atualizado."`
</ParamField>

<ParamField path="file" type="string">
  Nova URL ou Base64 do arquivo. <br />
  Obrigatório se `type` for mídia. <br />
  Exemplo: `"https://exemplo.com/novo-arquivo.jpg"`
</ParamField>

<ParamField path="docName" type="string">
  Novo nome para arquivo de documento. <br />
  Opcional, usado apenas com `type: document`. <br />
  Exemplo: `"catalogo-2024.pdf"`
</ParamField>

<Info>
  **Importante:** Templates criados pelo WhatsApp não podem ser editados. Apenas
  templates criados pela API são editáveis.
</Info>

<RequestExample>
```bash cURL
curl -X POST https://zapclass.com.br/api/quickreply/edit \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer zc_6FrX2qjM4bHT9pR6cC0sD8wLkG3nE2tZfV5uN1aP" \
  -d '{
    "id": "rb9da9c03637452",
    "text": "Olá! Nosso horário foi atualizado: Seg-Sex 8h-20h"
  }'
```

```javascript JavaScript
const response = await fetch("https://zapclass.com.br/api/quickreply/edit", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    Authorization: "Bearer zc_6FrX2qjM4bHT9pR6cC0sD8wLkG3nE2tZfV5uN1aP",
  },
  body: JSON.stringify({
    id: "rb9da9c03637452",
    text: "Olá! Nosso horário foi atualizado: Seg-Sex 8h-20h",
  }),
});

const data = await response.json();
if (data.success) {
  console.log("Template atualizado:", data.quickReply.id);
}
```

```php PHP
<?php
$ch = curl_init('https://zapclass.com.br/api/quickreply/edit');

$payload = [
    "id" => "rb9da9c03637452",
    "text" => "Olá! Nosso horário foi atualizado: Seg-Sex 8h-20h"
];

curl_setopt($ch, CURLOPT_POST, 1);
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode($payload));
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    'Content-Type: application/json',
    'Authorization: Bearer zc_6FrX2qjM4bHT9pR6cC0sD8wLkG3nE2tZfV5uN1aP'
]);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

$response = curl_exec($ch);
$data = json_decode($response, true);

if ($data['success']) {
    echo "Template atualizado: {$data['quickReply']['id']}\n";
} else {
    echo "Erro: " . $data['message'];
}

curl_close($ch);
?>
```

</RequestExample>
